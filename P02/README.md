
# P02: Afegint la documentació de seguretat al repositori

## Breu descripció

Al llarg d’aquest projecte heu estat documentant una sèrie de guies. Però de poc en serveix fer guies tècniques en documents si després no són fàcilment accessibles. Es crea un problema que s’anomena **data silo**.

Per evitar-lo, aprofitarem que tot just hem començat a treballar amb el control de versions (Git i GitHub) i crearem un repositori que inclourà la documentació en format Markdown.

---

## Tasques a realitzar

- Crear un repositori que es digui **Projecte2**.
- Crear un fitxer **README.md** que presenti el projecte.  
  - Utilitzeu recursos de Markdown per fer una descripció visualment atractiva.
  
- Dins d’aquest projecte:
  - Crear una carpeta per les **tasques 2 i 3**.

- A cada carpeta:
  - Crear un fitxer **README.md** explicant l’activitat i el contingut de la carpeta (arxiu `solucio.md`).
  - Afegir un **hipervincle** per accedir directament a l’arxiu `solucio.md`.

- Crear una carpeta **img** a cadascuna de les dues carpetes:
  - Pujar les imatges de les captures de pantalla.
  - Assegurar que:
    - Tenen una mida adequada.
    - Estan en format **PNG**.

- Crear a cada repositori un arxiu anomenat **solucio.md**:
  - Documentar la tasca en format Markdown.
  - Incloure les imatges dins del document.

- Assegurar que:
  - Cada imatge inclou una **descripció alternativa** (atribut `alt`).
